摘要（Abstract）

散列表，也叫哈希表，就是通过计算一个关于键值的函数，将所需查询的数据映射到表中的一个位置来访问记录，这加快了查找速度。

他的实现算法分为两步：

  1. 是用 hash Function将被查找的键转化为数组的一个索引；

  2. collision-resolution process ，当多个键对应一个索引的时候，我们如何处理；

使用 Hash table 在一般应用中拥有（均摊后）常数级别的查找 和插入操作的符号表！

但他的储存是无序 的，一有序性相关的操作是运行旱都将会是线性的。

hash Functions 的实现

散列函数的计算就，就是将键转化为数组的索引。

对于每种类型的键我们都需要一个与这对应的散列函数。

在java中，所有的数据类型都继承 了一个能够返回一个32-bit的integer 的 hashCode() ,当我们重写它的时候必须要保证他们equals()的一致性。

自定义hashCode（）的时候，我们要尽力的将键平均散布为所有可能的32位整数。这也是散列表算法分析前提，即均匀散列假设：

我们这使用的散列函数能够均匀并独立地将所有的键散布于0到M-1之间。
