背景知识

Java中 String 的主要性质：

  1. 不可变性

  2. 获得索引charAt()、长度length() 、子字符串substring() 可以在常数时间内完成

  3. 而字符串的转接（+） 所用的时间秘结果字符串的长度成正比。

键索引计数法

键索引计数法排序N个键为0到R-1的整数的元素需要访问数组11N+4R+1次

键索引计数法对小整数键排序非常有效，其突破了NlogN的排序算法运行时间下限，它是常数级别。

键索引计数法对字符串数组的排序是稳定的。这是它十分重要的特点。

低位优先的字符串排序

实现

上面的低位优先的字符串排序的应用场景是定长字符串；其依赖于键索引计数法的稳定性。

低位优先的字符串排序算法能够稳定地将定长字符串排序。对了基于R个字符的字母表的N个以长为W的字符串为键的元素，低位优先的字符串排序 要访问~7WN+3W次数组，使用的额外空间与N+R成正比。

低位优先的字符串排序意义重大：它是一种适用于一般应用的线性时间排序算法。——运行时间与回个话的规模成正比。

高位优先的排序

MSD中的几个陷阱：

  1. 小型子数组 小型子数组对于高位优先的字符串排序的性能至关重要，小的数组会极大的占用性能。 将小数组切换到到插入排序对高位优先的排序算法是必须的！！！ 这种情况在其他的递归算法中也是必须的。

  2. 等值键问题 MSD最坏的情况就是所有的键均相同，同样大量含有相同前缀也会产生同样的问题。

性能的问题

对随机输入，算法的运行时间是亚线性的。 非随机输入，算法的时间是接近线性的。

在将长字符串排序时，令MSD自算法发挥出最大效率的主要挑战在于处理数据中的非随机因素
