摘要

文件读写open()

  • 使用Python内置的open()函数，传入文件名和标示符:f = open('/Users/michael/test.txt', 'r')
  • 如果文件不存在，open()函数就会抛出一个IOError的错误.
  • 调用open()函数read()方法可以一次读取文件的全部内容，Python把内容读到内存，用一个str对象表示.
  • 最后一步是调用close()方法关闭文件。文件使用完毕后必须关闭，因为文件对象会占用操作系统的资源，并且操作系统同一时间能打开的文件数量也是有限的。

在实际中为了保证无论是否出错都能正确地关闭文件，我们可以使用with语句来自动帮我们调用close()方法：(好习惯)

ps:
调用read()会一次性读取文件的全部内容，如果文件有10G，内存就爆了，所以，要保险起见，可以反复调用read(size)方法，每次最多读取size个字节的内容。另外，调用readline()可以每次读取一行内容，调用readlines()一次读取所有内容并按行返回list。

几个标示符

r默认都是读取文本文件，并且是UTF-8编码的文本文件
rb读取二进制文件
w写文本文件
wb表示写二进制文件.

几个其他的参数：
encoding='gbk' 读取GBK编码的文件（也可以是别的）
errors='ignore' 遇到UnicodeDecodeError，因为在文本文件中可能夹杂了一些非法编码的字符。遇到这种情况，设置errors参数，表示如果遇到编码错误后如何处理：最简单的方式是直接忽略

StringIO

tringIO顾名思义就是在内存中读写str。
getvalue()方法用于获得写入后的str。

BytesIO

在内存中读写二进制数据
请注意，写入的不是str，而是经过UTF-8编码的bytes。

操作文件和目录

os模块可以直接调用操作系统提供的接口函数
获取详细的系统信息:可以调用uname()函数
操作系统中定义的环境变量: 全部保存在os.environ中
获取某个环境变量的值: 调用os.environ.get('key')

操作文件和目录的函数一部分放在os模块中，一部分放在os.path模块中，这一点要注意一下。

序列化

变量从内存中变成可存储或传输的过程称之为序列化，在Python中叫pickling，在其他语言中也被称之为serialization，marshalling，flattening等等，都是一个意思。
Python提供了pickle模块来实现序列化。
pickle.dumps()方法把任意对象序列化成一个bytes，然后，就可以把这个bytes写入文件pickle.dump()直接把对象序列化后写入一个file-like Object：

把对象从磁盘读到内存时，可以先把内容读到一个bytes，然后用pickle.loads()方法反序列化出对象，也可以直接用pickle.load()方法从一个file-like Object中直接反序列化出对象。

JSON

不同的编程语言之间传递对象，就必须把对象序列化为标准格式，比如XML.JSON
JSON标准规定JSON编码是UTF-8.
Python内置的json模块提供了非常完善的Python对象到JSON格式的转换。
json.dumps()方法返回一个str，内容就是标准的JSON。类似的，
json.dump()方法可以直接把JSON写入一个file-like Object
