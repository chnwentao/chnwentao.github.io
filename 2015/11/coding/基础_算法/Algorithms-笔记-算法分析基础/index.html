摘要（Abstract）

本文摘要：介绍了一如何进行算法分析，介绍了一下科学方法的原则；数量级的概念；数学模型的步骤；近似的概念；倍率实验（待研究）；对输入依赖的处理；内存占用的分析。

Knuth 证明了原则上我们只要正确并完整的使用了下面写的一方法就能够对程度作出详细，准备的预测。

一个程序的运行的总时间和两点有关：

  • 执行每条语句的耗时；——取决于编译器和操作系统

  • 执行每条语句的频率； —— 取决于程序本身和输入；

背景知识

科学方法的原则：

我们所设计的实验（分析算法的）必须是可重现的；
我们所以的假设也必须是可证伪的；

数学模型

建立的步骤

  1. 确成输入模型

  2. 识别内循环
    内循环：执行最频繁的指令；内循环决定了程序 执行的总时间，

  3. 根据内循环中的操作确定成本模型；

近似的概念

计算数量级的时候以最记数量级为准，忽略低数量级的。

增长级数

有的书叫算法的时间复杂度
详细的算法分析可能会用所有的人类发明的数学工具，这是专家的事儿，我们只要掌握几种常用的就好：

常数级别

普通语句 如 a = b + c;

对数级别

logN 二分策略 如： 二分查找
数学上：对娄的底数和增长的数量级无关，所以我们在说明对数级别时一般使用logN.

线性级别

N 一重循环 如：for (int i = 0; i < N ; i++) { 。。。。。。。}

线性对数级别

NlogN 分治 如：归并排序

平方级别

N2 双层循环

立方级别

N3 三层循环

指数级别

2N 穷举查找
指数级别的算法非常慢——不能用它们解决大规模的问题；

倍率实验

倍率实验是一种估计运行时间的增长数量级的简单方法；
在有性能压力的情况下应该考虑对编写过的所有程序进行倍率实验。
他可以用来：

  • 评估程序解决大型问题的可能性

  • 评估使用更快的计算机所产生的价值
    一般来说：新的计算机比老的快X倍，那运行时间就是原来的1/x。但是这个对平方或立方级别的算法不适用，

随机化算法

内存分析

比算法分析简单多了，总的内存占用 = 变量的数量 * 他们的对应占的字节数。

原始数据的占用量：

一个字节 = 8位
int、float = 4个字节
double/long = 8个字节
char = 2个字节
boolean/byte = 1个字节

对象的占用量：

对象的占用量 =对象开销（16byte）+ 域（int(4B)/String等的引用（8B））+ 填充字节（4B） + 他引用的所有对象的占用

内部类对象(如链表)的占用量

内部类对象(如链表)的占用量= 对象开销（16byte）+ 额外开销（8B）+ 域（int(4B)/String等的引用（8B））+ 他引用的所有对象的占用
额外开销（8B）是一个指向外部类的引用；

数组的占用量：

数组的占用量： = 24B的头信息（=对象开销（16B）+ 长度信息（4B）+ 填充字节（4B)） + 域（int(4B)/String等的引用（8B）+ 他引用的所有对象的占用

字符串对象的占用量：

Java中一个字符串对象中包含的字符串的值是一个引用，而不一个真正的字符串的拷贝，java中是多个字符串共用一个字符数组的。
同理，字符串的子串和字符串不但共用一个字符数组。对象中包含的字符串的引用也是一样的，就是偏移量不同而已。
字符串对象的占用量 =对象开销（16byte）+ 指向字符数组的引用（8B） + 偏移量(4B） +度信息（4B）+ Hash值(4B） + 填充字节（4B） + 他引用的字符数组对象的占用

扩展：
在递归中创建数组或其他的对象是很危险的，——这意味着每一次递归调用都会使用大量的内存。
